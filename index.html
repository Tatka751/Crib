Перелік основних команд
Команда Опис
git config --global user.name blabla користувач
git config --global user.email blabal@bla його пошта
git init ініціалізація в каталозі проекту
git status переглянути статус файлів.каталогів проекту
git add додати файли для індексації
git commit закомітити зміни
git log продивитись логи
git checkout переміщення між гілками проекту
git branch створення, перегляд, перейменування, видалення гілки проекту
git push вивантаження з локального репозиторія у віддалений
git pull завнатаження з віддаленого репозиотрію в локальний
git merge злиття кількох бранчів в один
git clone клонування існуючого репозиторію
.gitignore файл це файл, в якому перелічені файли, розширення/розтаушвання файлів,
які повинні ігноруватися. Його розміщають в рлбочому каталозі
git revert відміна зроблених змін, але не способом їх скидання(видалення) як reset,
а способом додавання нового коміту, який відміняє непотрібні попередні
до правильного коміту
git reset команда "скидання" стану репозиторія до визначеного коміту. При цьому дані
репозиторію фізично офоновлюються на всі трьох рівнях - історія комітів,
проіндексовані файли и робочий каталогом
git rebase перенесення групи комітів. Беруться всі коміти із вибраної гілки і застосовуються
до вибраної гілки, тобто коміти переприсвоюються і гіт створює нові коміти
git cherry-pick команда, за допомогою якої можна застосувати вибрані коміти до бранча,
або відмінити якісь коміти з поточної версії гілки
git stash тимчасове приховання змін в робочій версії, коли ще не прийшов час оформити ці зміни в коміт
